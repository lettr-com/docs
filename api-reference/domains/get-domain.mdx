# Get Domain

Retrieve details and verification status of a specific domain.

<ParamField path="domain" type="string" required>
  The domain name to retrieve.
</ParamField>

## Request

<CodeGroup>
```bash cURL
curl https://app.uselettr.com/api/domains/secure.example.com \
  -H "Authorization: Bearer le_xxxxxxxxxxxx"
```

```javascript Node.js
const domain = await lettr.domains.get('secure.example.com');
```

```php PHP
$domain = $lettr->domains->get('secure.example.com');
```

```python Python
domain = lettr.domains.get('secure.example.com')
```
</CodeGroup>

## Response

```json
{
  "domain": "secure.example.com",
  "cname_status": "valid",
  "compliance_status": "valid",
  "dkim_status": "valid",
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-01-15T10:35:00Z"
}
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `domain` | string | The domain name |
| `cname_status` | string | CNAME record verification status |
| `compliance_status` | string | Compliance verification status |
| `dkim_status` | string | DKIM record verification status |
| `created_at` | string \| null | ISO 8601 timestamp when the domain was created |
| `updated_at` | string \| null | ISO 8601 timestamp when the domain was last updated |

## Status Values

| Status | Description |
|--------|-------------|
| `valid` | Record is properly configured and verified |
| `pending` | Verification is in progress |
| `invalid` | Record is missing or misconfigured |
| `unverified` | Record has not been verified yet |

## Errors

| Code | Description |
|------|-------------|
| `not_found` | Domain not found in your account |
