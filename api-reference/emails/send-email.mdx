# Send Email

Send an email to one or more recipients.

<Warning>
  The total email content (text + HTML + attachments) is limited to 20 MB.
</Warning>

<ParamField body="from" type="string" required>
  Sender email address. Must be from a verified domain. Maximum 255 characters.
</ParamField>

<ParamField body="from_name" type="string">
  Friendly sender name. Maximum 255 characters.
</ParamField>

<ParamField body="subject" type="string" required>
  Email subject line. Maximum 998 characters.
</ParamField>

<ParamField body="to" type="string | string[]" required>
  Recipient email address(es). Can be a single email string or an array of email addresses. Maximum 50 recipients. Each email address has a maximum of 255 characters.
</ParamField>

<ParamField body="html" type="string">
  HTML content of the email. Required if `text` is not provided. Expected in the UTF-8 charset.
</ParamField>

<ParamField body="text" type="string">
  Plain text content of the email. Required if `html` is not provided. Expected in the UTF-8 charset.
</ParamField>

<ParamField body="amp_html" type="string">
  AMPHTML Email content for the email's `text/x-amp-html` MIME part. At least one of `html` or `text` must also be provided. Expected in the UTF-8 charset.
</ParamField>

<ParamField body="cc" type="string | string[]">
  CC (Carbon Copy) recipient address(es). Can be a single email string or an array.
</ParamField>

<ParamField body="bcc" type="string | string[]">
  BCC (Blind Carbon Copy) recipient address(es). Can be a single email string or an array.
</ParamField>

<ParamField body="reply_to" type="string">
  Reply-to email address. Maximum 255 characters.
</ParamField>

<ParamField body="reply_to_name" type="string">
  Reply-to display name. Maximum 255 characters.
</ParamField>

<ParamField body="attachments" type="object[]">
  Array of file attachments.

  <Expandable title="Attachment object">
    <ParamField body="name" type="string" required>
      Name of the file. Maximum 255 characters.
    </ParamField>
    <ParamField body="data" type="string" required>
      The content of the attachment as a Base64 encoded string. The string must not contain `\r\n` line breaks.
    </ParamField>
    <ParamField body="type" type="string" required>
      MIME type of the file. Maximum 255 characters.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="options" type="object">
  Email sending options.

  <Expandable title="Options object">
    <ParamField body="open_tracking" type="boolean" default="true">
      Enable open tracking pixel.
    </ParamField>
    <ParamField body="click_tracking" type="boolean" default="true">
      Enable click tracking for links in the email.
    </ParamField>
    <ParamField body="transactional" type="boolean" default="true">
      Mark this email as transactional.
    </ParamField>
    <ParamField body="inline_css" type="boolean" default="false">
      Automatically inline CSS styles in the HTML content.
    </ParamField>
    <ParamField body="perform_substitutions" type="boolean" default="true">
      Enable or disable substitutions of variables in email content.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="campaign_id" type="string">
  Associate this email with a campaign for analytics purposes. Maximum 64 characters.
</ParamField>

<ParamField body="metadata" type="object">
  Custom metadata as key-value pairs. Values must be strings. Metadata is available in Webhook events.
</ParamField>

<ParamField body="substitution_data" type="object">
  Custom variables as key-value pairs. Values must be strings. These are passed to email templates and replace placeholders in email content.
</ParamField>

## Request

```bash
curl -X POST https://app.uselettr.com/api/emails \
  -H "Authorization: Bearer le_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "you@example.com",
    "to": ["recipient@example.com"],
    "subject": "Hello World",
    "html": "<p>Welcome to Lettr!</p>"
  }'
```

## Response

**Success (200)**

```json
{
  "success": true,
  "message": "Email queued for delivery.",
  "data": {
    "request_id": "7582751837467401763",
    "accepted": 1,
    "rejected": 0
  }
}
```

## Errors

All error responses follow the same format:

```json
{
  "success": false,
  "message": "Error description",
  "errors": []
}
```

**400 Bad Request**

Returned when the email fails to send due to configuration issues (e.g., unverified domain).

```json
{
  "success": false,
  "message": "Failed to send email.",
  "errors": ["Sending domain is not configured or verified."]
}
```

**422 Unprocessable Entity**

Returned when request validation fails.

```json
{
  "success": false,
  "message": "Validation failed.",
  "errors": {
    "from": ["The sender email address is required."]
  }
}
```

**502 Bad Gateway**

Returned when the email transmission fails due to an upstream error.

```json
{
  "success": false,
  "message": "Failed to send email.",
  "errors": ["Upstream service error"]
}
```
