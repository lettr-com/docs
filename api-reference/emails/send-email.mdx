# Send Email

Send an email to one or more recipients.

<ParamField body="from" type="string" required>
  Sender email address. Must be from a verified domain. Maximum 255 characters.
</ParamField>

<ParamField body="from_name" type="string">
  Friendly sender name. Maximum 255 characters.
</ParamField>

<ParamField body="subject" type="string" required>
  Email subject line. Maximum 998 characters.
</ParamField>

<ParamField body="to" type="string[] | object[]" required>
  Array of recipient email addresses. Maximum 50 recipients. Can be simple email strings or recipient objects.
  
  <Expandable title="Recipient object">
    <ParamField body="email" type="string" required>
      Recipient email address.
    </ParamField>
    <ParamField body="name" type="string">
      Recipient display name.
    </ParamField>
    <ParamField body="metadata" type="object">
      Recipient-specific metadata. Available in Webhooks events.
    </ParamField>
    <ParamField body="substitution_data" type="object">
      Recipient-specific substitution_data. Passed to email templates and replaces placeholders in email content.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="html" type="string">
  HTML content of the email. Required if `text` is not provided. Expected in the UTF-8 charset.
</ParamField>

<ParamField body="text" type="string">
  Plain text content of the email. Required if `html` is not provided. Expected in the UTF-8 charset.
</ParamField>

<ParamField body="amp_html" type="string">
  AMPHTML Email content for the email's `text/x-amp-html` MIME part. At least one of `html` or `text` must also be provided. Expected in the UTF-8 charset.
</ParamField>

<ParamField body="cc" type="string[]">
  Array of CC (Carbon Copy) recipient addresses.
</ParamField>

<ParamField body="bcc" type="string[]">
  Array of BCC (Blind Carbon Copy) recipient addresses.
</ParamField>

<ParamField body="reply_to" type="string">
  Reply-to email address.
</ParamField>

<ParamField body="attachments" type="object[]">
  Array of file attachments.
  
  <Expandable title="Attachment object">
    <ParamField body="name" type="string" required>
      Name of the file.
    </ParamField>
    <ParamField body="content" type="string">
      The content of the attachment as a Base64 encoded string. The string should not contain \r\n line breaks. At least one of `remote_path` or `content` must be provided.
    </ParamField>
    <ParamField body="type" type="string">
      MIME type of the file.
    </ParamField>
    <ParamField body="remote_path" type="string">
      TBD Include the remote_path parameter to send attachments from a remote file. This parameter accepts a URL to the file you want to attach. At least one of `remote_path` or `content` must be provided.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="headers" type="object">
  Custom email headers as key-value pairs.
</ParamField>

<ParamField body="options" type="object">
  Email sending options.
  
  <Expandable title="Options object">
    <ParamField body="open_tracking" type="boolean" default="true">
      Enable open tracking pixel.
    </ParamField>
    <ParamField body="click_tracking" type="boolean" default="true">
      Enable click tracking for links in the email.
    </ParamField>
    <ParamField body="transactional" type="boolean" default="true">
      Mark this email as transactional.
    </ParamField>
    <ParamField body="inline_css" type="boolean" default="false">
      Automatically inline CSS styles in the HTML content.
    </ParamField>
    <ParamField body="perform_substitutions" type="boolean" default="true">
      Enable or disable substitutions of variables in email content.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="campaign_id" type="string">
  Associate this email with a campaign for analytics purposes.
</ParamField>

<ParamField body="metadata" type="object">
  Custom metadata as key-value pairs. Values must be strings. Metadata is available in Webhook events.
</ParamField>

<ParamField body="substitution_data" type="object">
  Custom variables as key-value pairs. Those are passed to email templates and replaces placeholders in email content.
</ParamField>

## Request

<CodeGroup>
```bash cURL
curl -X POST https://app.uselettr.com/api/emails \
  -H "Authorization: Bearer le_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "you@example.com",
    "to": ["recipient@example.com"],
    "subject": "Hello World",
    "html": "<p>Welcome to Lettr!</p>"
  }'
```

```javascript Node.js
const email = await lettr.emails.send({
  from: 'you@example.com',
  to: ['recipient@example.com'],
  subject: 'Hello World',
  html: '<p>Welcome to Lettr!</p>'
});
```

```php PHP
$email = $lettr->emails->send([
    'from' => 'you@example.com',
    'to' => ['recipient@example.com'],
    'subject' => 'Hello World',
    'html' => '<p>Welcome to Lettr!</p>'
]);
```

```python Python
email = lettr.emails.send(
    from_email='you@example.com',
    to=['recipient@example.com'],
    subject='Hello World',
    html='<p>Welcome to Lettr!</p>'
)
```
</CodeGroup>

## Errors

| Code | Description |
|------|-------------|
| `invalid_from` | Sender address not from verified domain |
| `invalid_to` | Invalid recipient email address |
| `suppressed` | Recipient is suppressed |
| `rate_limit_exceeded` | Too many requests |
