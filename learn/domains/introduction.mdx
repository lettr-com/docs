# Managing Domains

Domains are the foundation of your email infrastructure in Lettr. Properly configured domains ensure your emails reach the inbox, maintain your brand identity, and comply with email authentication standards.

<Warning>
  Domain not verifying? Check our [troubleshooting guide](/knowledge-base/troubleshooting/authentication) for common DNS issues.
</Warning>

## Types of Domains

Lettr supports four types of domains, each serving a specific purpose:

## Domain Comparison

| Domain Type | Purpose | Required DNS Records |
|-------------|---------|---------------------|
| Sending | Send emails from your domain | SPF (TXT) or CNAME, DKIM (CNAME), DMARC |
| Tracking | Custom tracking links | CNAME (SSL), CNAME (proxy) |
| Inbound | Receive emails | MX records pointing to Lettr |
| Storage | Host email images | CNAME (SSL), CNAME (proxy) |

## Why Use Custom Domains?

Using your own domains instead of shared Lettr domains provides:

- **Brand Consistency** - All links and addresses use your domain
- **Improved Deliverability** - Authenticated emails from your domain build reputation
- **User Trust** - Recipients see familiar domains in links and sender addresses
- **Compliance** - Meet email authentication requirements (SPF, DKIM, DMARC)
- **Reputation Isolation** - Your sending reputation is separate from other senders

## Email Authentication

Email authentication protocols verify that emails are legitimately sent by the domain owner. Lettr requires proper authentication for all sending domains.

## Add Domain via API

```javascript
const domain = await lettr.domains.create({
  name: 'example.com',
  type: 'sending' // 'sending', 'tracking', 'inbound', or 'storage'
});

// Get DNS records to add
console.log(domain.dnsRecords);
```

## Domain Status

Domains can have different statuses during and after verification:

| Status | Description |
|--------|-------------|
| `not_started` | Domain added but verification not initiated |
| `pending` | DNS records not yet detected, verification in progress |
| `verified` | All records verified and domain is active |
| `failed` | Verification failed after 72 hours (check DNS records) |
| `temporary_failure` | Previously verified domain with DNS issues detected |

<Info>
  For `temporary_failure` status, Lettr will recheck DNS records for 72 hours. If records are detected, status returns to `verified`. Otherwise, it changes to `failed`.
</Info>

## Check Domain Status

```javascript
const domain = await lettr.domains.get('dom_123');

console.log({
  name: domain.name,
  type: domain.type,
  status: domain.status,
  spfStatus: domain.spfStatus,
  dkimStatus: domain.dkimStatus,
  dmarcStatus: domain.dmarcStatus
});
```

## Subdomains vs Root Domains

We recommend using subdomains for sending emails to isolate your sending reputation:

| Subdomain | Use Case |
|-----------|----------|
| `mail.example.com` | Transactional emails |
| `news.example.com` | Marketing newsletters |
| `updates.example.com` | Product updates |

<Tip>
  Using subdomains protects your root domain's reputation. If one subdomain experiences deliverability issues, your other subdomains remain unaffected.
</Tip>
