# Rate Limits

Lettr enforces rate limits to ensure fair usage and maintain service reliability for all users.

## API Rate Limits

| Limit Type | Value | Window |
|------------|-------|--------|
| API requests | 300 requests | 5 minutes |
| High-volume API | 2,000 requests | 5 minutes |
| Authentication failures | 5 failures | 5 minutes (15 min block) |

## Email Sending Limits

| Limit | Value |
|-------|-------|
| Recipients per request | 50 (combined to, cc, bcc) |
| Monthly email quota | Based on your plan |

## Rate Limit Response

When you exceed rate limits, you'll receive a `429 Too Many Requests` response:

```json
{
  "status": "error",
  "error_code": "rate_limit_exceeded",
  "message": "Too many requests. Please retry after 60 seconds.",
  "retry_after": 60
}
```

## Handling Rate Limits

Implement exponential backoff for rate limit errors:

```javascript
async function sendWithBackoff(emailData, maxRetries = 5) {
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      return await lettr.emails.send(emailData);
    } catch (error) {
      if (error.status === 429) {
        const retryAfter = error.retryAfter || Math.pow(2, attempt);
        console.log(`Rate limited. Retrying in ${retryAfter} seconds...`);
        await sleep(retryAfter * 1000);
      } else {
        throw error;
      }
    }
  }
  throw new Error('Max retries exceeded');
}
```

## Batch Sending with Rate Limiting

When sending large volumes, implement pacing:

```javascript
const recipients = [...]; // Large list
const batchSize = 50;
const delayBetweenBatches = 200; // milliseconds

for (let i = 0; i < recipients.length; i += batchSize) {
  const batch = recipients.slice(i, i + batchSize);

  await lettr.emails.send({
    from: 'you@example.com',
    to: batch,
    subject: 'Newsletter',
    html: content
  });

  // Pace your requests
  if (i + batchSize < recipients.length) {
    await sleep(delayBetweenBatches);
  }
}
```

## Checking Your Usage

Monitor your email usage via the dashboard or API:

```javascript
const usage = await lettr.usage.get();

console.log({
  sent: usage.emailsSent,
  quota: usage.monthlyQuota,
  remaining: usage.monthlyQuota - usage.emailsSent,
  percentUsed: (usage.emailsSent / usage.monthlyQuota) * 100
});
```
