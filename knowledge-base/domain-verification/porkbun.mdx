# Porkbun

This guide walks you through adding DNS records on Porkbun to verify your domain with Lettr.

<Tip>
  It is best practice to use a subdomain (e.g., `sender.example.com`) instead of the root domain. Using a subdomain allows for proper reputation segmentation and is especially important if receiving emails with Lettr.
</Tip>

## Add Domain to Lettr

First, log in to your [Lettr Dashboard](https://app.lettr.com) and go to **Domains â†’ Add Domain**. Enter your domain name and select the domain type.

## Log in to Porkbun

1. Log in to your [Porkbun account](https://porkbun.com/account/domainsSpeedy)
2. Find your domain in the list
3. Click **DNS** to manage DNS records

---

## Sending Domain Setup

Choose one of the following options based on whether you're using a subdomain or primary domain.

### Option 1: Subdomain (CNAME Verification)

If you're setting up a subdomain like `sender.example.com`, add the following records:

#### Bounce Domain CNAME

1. In the **DNS** section, select `CNAME` as the **Type**
2. Enter `sender` for the **Host** (or your chosen subdomain)
3. Enter `sparkpostmail.com` for the **Answer / Value**
4. Use the default **TTL** of `600`
5. Click **Add**

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `CNAME` |
| Host | Host | `sender` |
| Answer / Value | Value | `sparkpostmail.com` |
| TTL | - | `600` |

### Option 2: Primary Domain (TXT SPF Verification)

If you're using your root domain `example.com`, add an SPF record:

#### SPF Record

1. Select `TXT` as the **Type**
2. Leave **Host** empty (for root domain)
3. Enter `v=spf1 include:_spf.sparkpostmail.com ~all` for the **Answer / Value**
4. Use the default **TTL** of `600`
5. Click **Add**

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `TXT` |
| Host | Host | (empty for root) |
| Answer / Value | Value | `v=spf1 include:_spf.sparkpostmail.com ~all` |
| TTL | - | `600` |

<Info>
  If you already have an SPF record, merge Lettr's include with your existing record.
</Info>

### DKIM Record (Required for All Sending Domains)

1. Select `TXT` as the **Type**
2. Copy the **Host** from your Lettr dashboard (e.g., `scph0226._domainkey.sender`)
3. Copy the **Answer / Value** from your Lettr dashboard
4. Click **Add**

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `TXT` |
| Host | Host | `scph0226._domainkey.sender` |
| Answer / Value | Value | `v=DKIM1;k=rsa;h=sha256;p=MIGfMA0...` |
| TTL | - | `600` |

<Info>
  The DKIM hostname and value are unique to your domain. Copy them exactly from your Lettr dashboard.
</Info>

### DMARC Record (Required for All Sending Domains)

1. Select `TXT` as the **Type**
2. Enter `_dmarc.sender` for the **Host**
3. Enter `v=DMARC1;p=none;` for the **Answer / Value**
4. Click **Add**

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `TXT` |
| Host | Host | `_dmarc.sender` |
| Answer / Value | Value | `v=DMARC1;p=none;` |
| TTL | - | `600` |

## Tracking Domain Setup

To set up a custom tracking domain, add two CNAME records:

### SSL Certificate CNAME

1. Select `CNAME` as the **Type**
2. Copy the **Host** from your Lettr dashboard (e.g., `_acme-challenge.click.sender`)
3. Copy the **Answer / Value** from your Lettr dashboard
4. Click **Add**

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `CNAME` |
| Host | Host | `_acme-challenge.click.sender` |
| Answer / Value | Value | (provided in dashboard) |

### Traffic CNAME

1. Select `CNAME` as the **Type**
2. Enter your tracking subdomain for the **Host** (e.g., `click.sender`)
3. Enter `proxy.lettr-tracking.com` for the **Answer / Value**
4. Click **Add**

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `CNAME` |
| Host | Host | `click.sender` |
| Answer / Value | Value | `proxy.lettr-tracking.com` |

<Tip>
  SSL certificates are automatically generated and renewed by Lettr. All tracking URLs use HTTPS.
</Tip>

## Inbound Domain Setup

To receive emails at your domain, add MX records:

1. Select `MX` as the **Type**
2. Enter `inbound` for the **Host**
3. Enter `rx1.sparkpostmail.com` for the **Answer / Value**
4. Enter `10` for the **Priority**
5. Click **Add**

Repeat for the other two MX records:

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `MX` |
| Host | Host | `inbound` |
| Answer / Value | Value | `rx1.sparkpostmail.com` |
| Priority | Priority | `10` |

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `MX` |
| Host | Host | `inbound` |
| Answer / Value | Value | `rx2.sparkpostmail.com` |
| Priority | Priority | `10` |

| Porkbun | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `MX` |
| Host | Host | `inbound` |
| Answer / Value | Value | `rx3.sparkpostmail.com` |
| Priority | Priority | `10` |

<Warning>
  When you configure MX records, all emails sent to that domain/subdomain will be routed to Lettr. We recommend using a subdomain like `inbound.example.com`.
</Warning>

## Complete Verification

Return to your Lettr dashboard and click **Verify**. DNS propagation typically takes a few hours but is often faster.
