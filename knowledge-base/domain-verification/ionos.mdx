# IONOS

This guide walks you through adding DNS records on IONOS to verify your domain with Lettr.

<Tip>
  It is best practice to use a subdomain (e.g., `sender.example.com`) instead of the root domain. Using a subdomain allows for proper reputation segmentation and is especially important if receiving emails with Lettr.
</Tip>

## Add Domain to Lettr

First, log in to your [Lettr Dashboard](https://app.lettr.com) and go to **Domains â†’ Add Domain**. Enter your domain name and select the domain type.

## Log in to IONOS

1. Log in to your [IONOS account](https://my.ionos.com/domains)
2. Select your domain from the **Domain** list
3. Navigate to the **DNS** tab to manage DNS records

---

## Sending Domain Setup

Choose one of the following options based on whether you're using a subdomain or primary domain.

### Option 1: Subdomain (CNAME Verification)

If you're setting up a subdomain like `sender.example.com`, add the following records:

#### Bounce Domain CNAME

1. Click **Add record**
2. Select `CNAME` as the **Type**
3. Enter `sender` for the **Host name** (or your chosen subdomain)
4. Enter `sparkpostmail.com` for the **Points to**
5. Use the default **TTL** of `1 hour`
6. Click **Save**

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `CNAME` |
| Host name | Host | `sender` |
| Points to | Value | `sparkpostmail.com` |
| TTL | - | `1 hour` |

### Option 2: Primary Domain (TXT SPF Verification)

If you're using your root domain `example.com`, add an SPF record:

#### SPF Record

1. Click **Add record**
2. Select `TXT` as the **Type**
3. Enter `@` for the **Host name** (root domain)
4. Enter `v=spf1 include:_spf.sparkpostmail.com ~all` for the **Value**
5. Click **Save**

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `TXT` |
| Host name | Host | `@` |
| Value | Value | `v=spf1 include:_spf.sparkpostmail.com ~all` |
| TTL | - | `1 hour` |

<Info>
  If you already have an SPF record, merge Lettr's include with your existing record.
</Info>

### DKIM Record (Required for All Sending Domains)

1. Click **Add record**
2. Select `TXT` as the **Type**
3. Copy the **Host name** from your Lettr dashboard (e.g., `scph0226._domainkey.sender`)
4. Copy the **Value** from your Lettr dashboard
5. Click **Save**

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `TXT` |
| Host name | Host | `scph0226._domainkey.sender` |
| Value | Value | `v=DKIM1;k=rsa;h=sha256;p=MIGfMA0...` |
| TTL | - | `1 hour` |

<Info>
  The DKIM hostname and value are unique to your domain. Copy them exactly from your Lettr dashboard.
</Info>

### DMARC Record (Required for All Sending Domains)

1. Click **Add record**
2. Select `TXT` as the **Type**
3. Enter `_dmarc.sender` for the **Host name**
4. Enter `v=DMARC1;p=none;` for the **Value**
5. Click **Save**

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `TXT` |
| Host name | Host | `_dmarc.sender` |
| Value | Value | `v=DMARC1;p=none;` |
| TTL | - | `1 hour` |

## Tracking Domain Setup

To set up a custom tracking domain, add two CNAME records:

### SSL Certificate CNAME

1. Click **Add record**
2. Select `CNAME` as the **Type**
3. Copy the **Host name** from your Lettr dashboard (e.g., `_acme-challenge.click.sender`)
4. Copy the **Points to** from your Lettr dashboard
5. Click **Save**

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `CNAME` |
| Host name | Host | `_acme-challenge.click.sender` |
| Points to | Value | (provided in dashboard) |

### Traffic CNAME

1. Click **Add record**
2. Select `CNAME` as the **Type**
3. Enter your tracking subdomain for the **Host name** (e.g., `click.sender`)
4. Enter `proxy.lettr-tracking.com` for the **Points to**
5. Click **Save**

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `CNAME` |
| Host name | Host | `click.sender` |
| Points to | Value | `proxy.lettr-tracking.com` |

<Tip>
  SSL certificates are automatically generated and renewed by Lettr. All tracking URLs use HTTPS.
</Tip>

## Inbound Domain Setup

To receive emails at your domain, add MX records:

1. Click **Add record**
2. Select `MX` as the **Type**
3. Enter `inbound` for the **Host name**
4. Enter `rx1.sparkpostmail.com` for the **Points to**
5. Enter `10` for the **Priority**
6. Click **Save**

Repeat for the other two MX records:

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `MX` |
| Host name | Host | `inbound` |
| Points to | Value | `rx1.sparkpostmail.com` |
| Priority | Priority | `10` |

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `MX` |
| Host name | Host | `inbound` |
| Points to | Value | `rx2.sparkpostmail.com` |
| Priority | Priority | `10` |

| IONOS | Lettr | Example Value |
|-------|-------|---------------|
| Type | Type | `MX` |
| Host name | Host | `inbound` |
| Points to | Value | `rx3.sparkpostmail.com` |
| Priority | Priority | `10` |

<Warning>
  When you configure MX records, all emails sent to that domain/subdomain will be routed to Lettr. We recommend using a subdomain like `inbound.example.com`.
</Warning>

## Complete Verification

Return to your Lettr dashboard and click **Verify**. DNS propagation typically takes a few hours.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Records not detected">
    IONOS DNS may take up to 48 hours to propagate. Use [WhatsMyDNS](https://www.whatsmydns.net/) to check status.
  </Accordion>
  <Accordion title="Verification failed">
    Double-check record values for typos. Ensure the host name doesn't include your full domain.
  </Accordion>
</AccordionGroup>
