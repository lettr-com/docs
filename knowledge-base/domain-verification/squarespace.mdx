# Squarespace

This guide walks you through adding DNS records on Squarespace to verify your domain with Lettr.

<Tip>
  It is best practice to use a subdomain (e.g., `sender.example.com`) instead of the root domain. Using a subdomain allows for proper reputation segmentation and is especially important if receiving emails with Lettr.
</Tip>

## Add Domain to Lettr

Log in to your [Lettr Dashboard](https://app.lettr.com), go to **Domains**, open the tab for your domain type (e.g. **Sending Domains**), and click **+ New Sending Domain** (or the equivalent for Tracking, Inbound, or Storage). Enter your domain name.

## Log in to Squarespace

1. Log in to your [Squarespace domains page](https://account.squarespace.com/domains)
2. Click on your domain
3. Scroll down to the **Custom records** section

---

## Sending Domain Setup

Choose one of the following options based on whether you're using a subdomain or primary domain.

### Option 1: Subdomain (CNAME Verification)

If you're setting up a subdomain like `sender.example.com`, add the following records:

#### Bounce Domain CNAME

1. In the **Custom records** section, click **Add record**
2. Enter `sender` for the **Host** (or your chosen subdomain)
3. Select `CNAME` as the **Type**
4. Enter `sparkpostmail.com` for the **Data**
5. Use the default **TTL** of `4 hrs`
6. Click **Save**

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `CNAME` |
| Host | Host | `sender` |
| Data | Value | `sparkpostmail.com` |
| TTL | - | `4 hrs` |

### Option 2: Primary Domain (TXT SPF Verification)

If you're using your root domain `example.com`, add an SPF record:

#### SPF Record

1. Click **Add record**
2. Enter `@` for the **Host** (root domain)
3. Select `TXT` as the **Type**
4. Enter `v=spf1 include:_spf.sparkpostmail.com ~all` for the **Data**
5. Click **Save**

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `TXT` |
| Host | Host | `@` |
| Data | Value | `v=spf1 include:_spf.sparkpostmail.com ~all` |
| TTL | - | `4 hrs` |

<Info>
  If you already have an SPF record, merge Lettr's include with your existing record.
</Info>

### DKIM Record (Required for All Sending Domains)

1. Click **Add record**
2. Copy the **Host** from your Lettr dashboard (e.g., `scph0226._domainkey.sender`)
3. Select `TXT` as the **Type**
4. Copy the **Data** from your Lettr dashboard
5. Click **Save**

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `TXT` |
| Host | Host | `scph0226._domainkey.sender` |
| Data | Value | `v=DKIM1;k=rsa;h=sha256;p=MIGfMA0...` |
| TTL | - | `4 hrs` |

<Info>
  The DKIM hostname and value are unique to your domain. Copy them exactly from your Lettr dashboard.
</Info>

### DMARC Record (Required for All Sending Domains)

1. Click **Add record**
2. Enter `_dmarc.sender` for the **Host**
3. Select `TXT` as the **Type**
4. Enter `v=DMARC1;p=none;` for the **Data**
5. Click **Save**

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `TXT` |
| Host | Host | `_dmarc.sender` |
| Data | Value | `v=DMARC1;p=none;` |
| TTL | - | `4 hrs` |

## Tracking Domain Setup

To set up a custom tracking domain, add two CNAME records:

### SSL Certificate CNAME

1. Click **Add record**
2. Copy the **Host** from your Lettr dashboard (e.g., `_acme-challenge.click.sender`)
3. Select `CNAME` as the **Type**
4. Copy the **Data** from your Lettr dashboard
5. Click **Save**

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `CNAME` |
| Host | Host | `_acme-challenge.click.sender` |
| Data | Value | (provided in dashboard) |

### Traffic CNAME

1. Click **Add record**
2. Enter your tracking subdomain for the **Host** (e.g., `click.sender`)
3. Select `CNAME` as the **Type**
4. Enter `proxy.lettr-tracking.com` for the **Data**
5. Click **Save**

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `CNAME` |
| Host | Host | `click.sender` |
| Data | Value | `proxy.lettr-tracking.com` |

<Tip>
  SSL certificates are automatically generated and renewed by Lettr. All tracking URLs use HTTPS.
</Tip>

## Inbound Domain Setup

To receive emails at your domain, add MX records:

1. Click **Add record**
2. Enter `inbound` for the **Host**
3. Select `MX` as the **Type**
4. Enter `10` for the **Priority**
5. Enter `rx1.sparkpostmail.com` for the **Data**
6. Click **Save**

Repeat for the other two MX records:

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `MX` |
| Host | Host | `inbound` |
| Data | Value | `rx1.sparkpostmail.com` |
| Priority | Priority | `10` |

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `MX` |
| Host | Host | `inbound` |
| Data | Value | `rx2.sparkpostmail.com` |
| Priority | Priority | `10` |

| Squarespace | Lettr | Example Value |
|-------------|-------|---------------|
| Type | Type | `MX` |
| Host | Host | `inbound` |
| Data | Value | `rx3.sparkpostmail.com` |
| Priority | Priority | `10` |

<Warning>
  When you configure MX records, all emails sent to that domain/subdomain will be routed to Lettr. We recommend using a subdomain like `inbound.example.com`.
</Warning>

## Complete Verification

Return to your Lettr dashboard and click **Verify**. DNS propagation may take up to 72 hours but is often faster.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Records not detected">
    Squarespace DNS may take up to 72 hours to propagate. Use [WhatsMyDNS](https://www.whatsmydns.net/) to check status.
  </Accordion>
  <Accordion title="Can't find Custom records section">
    Make sure your domain is using Squarespace's nameservers. If using external DNS, manage records there instead.
  </Accordion>
</AccordionGroup>
