# Verify Your Domain on Dynadot

This guide walks you through adding DNS records for Lettr on Dynadot.

## Prerequisites

- A domain registered with Dynadot
- Access to your Dynadot account
- A Lettr account with your domain added

## Step 1: Add Domain to Lettr

1. Log in to your [Lettr dashboard](https://app.lettr.com)
2. Navigate to **Domains**
3. Click **Add Domain**
4. Enter your domain name and select the domain type
5. Copy the DNS records provided

## Step 2: Access Dynadot DNS

1. Log in to your [Dynadot account](https://www.dynadot.com)
2. Go to **My Domains** â†’ **Manage Domains**
3. Click on your domain name
4. Select the **DNS** tab
5. Choose **Dynadot DNS** if not already selected

<Info>
  Make sure your domain is set to use Dynadot's nameservers for these settings to work.
</Info>

## Step 3: Sending Domain Setup

Choose the setup that matches your needs:

### Option A: Subdomain (Recommended)

Use a subdomain like `sender.example.com` for sending emails.

#### Bounce Domain CNAME

1. In the **DNS Records** section, find an empty row or click **Add DNS Record**
2. Select **CNAME** as the **Record Type**
3. Enter `sender` in the **Subdomain** field (or your chosen subdomain)
4. Enter `sparkpostmail.com` in the **IP Address or Target Host** field
5. Click **Save DNS**

| Dynadot | Lettr | Example Value |
|---------|-------|---------------|
| Record Type | Type | `CNAME` |
| Subdomain | Host | `sender` |
| IP Address or Target Host | Value | `sparkpostmail.com` |

### Option B: Primary Domain

Use your root domain with TXT SPF verification.

#### SPF Record

1. In the **DNS Records** section, find an empty row or click **Add DNS Record**
2. Select **TXT** as the **Record Type**
3. Leave the **Subdomain** field empty (for root domain)
4. Enter `v=spf1 include:_spf.sparkpostmail.com ~all` in the **Text** field
5. Click **Save DNS**

| Dynadot | Lettr | Example Value |
|---------|-------|---------------|
| Record Type | Type | `TXT` |
| Subdomain | Host | (empty) |
| Text | Value | `v=spf1 include:_spf.sparkpostmail.com ~all` |

<Warning>
  If you already have an SPF record, merge it by adding `include:_spf.sparkpostmail.com` before the `~all` or `-all` mechanism.
</Warning>

### DKIM Record (Required for both options)

1. Click **Add DNS Record**
2. Select **TXT** as the **Record Type**
3. Enter the DKIM hostname from your Lettr dashboard (e.g., `scph0226._domainkey.sender`)
4. Enter the DKIM value from your dashboard in the **Text** field
5. Click **Save DNS**

| Dynadot | Lettr | Example Value |
|---------|-------|---------------|
| Record Type | Type | `TXT` |
| Subdomain | Host | `scph0226._domainkey.sender` |
| Text | Value | `v=DKIM1;k=rsa;h=sha256;p=MIGf...` |

### DMARC Record (Required for both options)

1. Click **Add DNS Record**
2. Select **TXT** as the **Record Type**
3. Enter `_dmarc.sender` (subdomain) or `_dmarc` (root domain) in the **Subdomain** field
4. Enter `v=DMARC1;p=none;` in the **Text** field
5. Click **Save DNS**

| Dynadot | Lettr | Example Value |
|---------|-------|---------------|
| Record Type | Type | `TXT` |
| Subdomain | Host | `_dmarc.sender` or `_dmarc` |
| Text | Value | `v=DMARC1;p=none;` |

## Step 4: Tracking Domain Setup

Custom tracking domains require two CNAME records.

### SSL Certificate CNAME

1. Click **Add DNS Record**
2. Select **CNAME** as the **Record Type**
3. Enter the SSL verification hostname from your dashboard (e.g., `_acme-challenge.click.sender`)
4. Enter the SSL verification value from your dashboard
5. Click **Save DNS**

### Traffic CNAME

1. Click **Add DNS Record**
2. Select **CNAME** as the **Record Type**
3. Enter your tracking subdomain (e.g., `click.sender`)
4. Enter `proxy.lettr-tracking.com` in the **IP Address or Target Host** field
5. Click **Save DNS**

| Dynadot | Lettr | Example Value |
|---------|-------|---------------|
| Record Type | Type | `CNAME` |
| Subdomain | Host | `click.sender` |
| IP Address or Target Host | Value | `proxy.lettr-tracking.com` |

<Tip>
  SSL certificates are automatically generated and renewed by Lettr.
</Tip>

## Step 5: Inbound Domain Setup

To receive emails at your domain, add MX records.

1. Click **Add DNS Record**
2. Select **MX** as the **Record Type**
3. Enter `inbound` in the **Subdomain** field (or your chosen subdomain)
4. Enter `rx1.sparkpostmail.com` in the **Target Host** field
5. Enter `10` for the **Distance/Priority**
6. Click **Save DNS**
7. Repeat for `rx2.sparkpostmail.com` and `rx3.sparkpostmail.com`

| Dynadot Subdomain | Priority | Target Host |
|-------------------|----------|-------------|
| `inbound` | 10 | `rx1.sparkpostmail.com` |
| `inbound` | 10 | `rx2.sparkpostmail.com` |
| `inbound` | 10 | `rx3.sparkpostmail.com` |

<Warning>
  Use a subdomain like `inbound.example.com` for receiving emails. Using your root domain will route ALL email to Lettr.
</Warning>

## Step 6: Complete Verification

1. Return to your [Lettr dashboard](https://app.lettr.com)
2. Navigate to **Domains**
3. Click **Verify** next to your domain
4. Wait for DNS propagation (usually 15-60 minutes for Dynadot)

## Troubleshooting

<AccordionGroup>
  <Accordion title="DNS changes not appearing">
    Dynadot DNS changes can take 15-60 minutes to propagate. If records aren't showing:
    - Verify you clicked **Save DNS** after adding records
    - Ensure your domain is using Dynadot nameservers
    - Check for typos in record values
  </Accordion>
  <Accordion title="Domain not using Dynadot DNS">
    If your domain uses external nameservers, you need to either:
    - Switch to Dynadot nameservers, or
    - Add DNS records at your current DNS provider
  </Accordion>
  <Accordion title="CNAME at root domain">
    Dynadot doesn't support CNAME records at the root domain. Use the subdomain option or TXT SPF verification for root domains.
  </Accordion>
  <Accordion title="Long TXT records">
    If you're having trouble with long DKIM values, try entering the value without any line breaks or extra spaces.
  </Accordion>
</AccordionGroup>
