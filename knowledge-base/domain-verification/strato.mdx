# Strato

This guide walks you through adding DNS records on Strato to verify your domain with Lettr.

<Tip>
  It is best practice to use a subdomain (e.g., `sender.example.com`) instead of the root domain. Using a subdomain allows for proper reputation segmentation and is especially important if receiving emails with Lettr.
</Tip>

## Add Domain to Lettr

Log in to your [Lettr Dashboard](https://app.lettr.com), go to **Domains**, open the tab for your domain type (e.g. **Sending Domains**), and click **+ New Sending Domain** (or the equivalent for Tracking, Inbound, or Storage). Enter your domain name.

## Log in to Strato

1. Log in to your [Strato account](https://www.strato.de/apps/CustomerService)
2. In the left navigation, go to **Domains â†’ Manage Domain**
3. Select your domain and navigate to settings

<Info>
  By default, Strato domains use Strato's mail server. For sending domains, you may need to create a subdomain and configure MX records there to avoid conflicts.
</Info>

## Sending Domain Setup

Choose one of the following options based on whether you're using a subdomain or primary domain.

### Option 1: Subdomain (CNAME Verification)

For subdomains, you'll need to first create the subdomain:

1. Create a new subdomain named `sender` (or your chosen name)
2. Navigate to the subdomain's settings
3. Go to the **DNS** tab

#### Bounce Domain CNAME

1. Click **manage** on the CNAME section
2. Enter `sparkpostmail.com` for the **Value**
3. Save the settings

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `sender` |
| Value | Value | `sparkpostmail.com` |

### Option 2: Primary Domain (TXT SPF Verification)

On the base domain settings:

1. Go to the **DNS** tab
2. Click **Manage TXT and CNAME records**

#### SPF Record

1. Click **Create another record**
2. Select `TXT` as the **Type**
3. Enter `@` for the **Name** (or leave empty for root)
4. Enter `v=spf1 include:_spf.sparkpostmail.com ~all` for the **Value**
5. Save the settings

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `@` |
| Value | Value | `v=spf1 include:_spf.sparkpostmail.com ~all` |

<Info>
  If you already have an SPF record, merge Lettr's include with your existing record.
</Info>

### DKIM Record (Required for All Sending Domains)

1. Click **Create another record**
2. Select `TXT` as the **Type**
3. Copy the **Name** from your Lettr dashboard (e.g., `scph0226._domainkey.sender`)
4. Copy the **Value** from your Lettr dashboard
5. Save the settings

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `scph0226._domainkey.sender` |
| Value | Value | `v=DKIM1;k=rsa;h=sha256;p=MIGfMA0...` |

<Info>
  DKIM records are case-sensitive. Copy the value exactly using the copy icon in your Lettr dashboard.
</Info>

### DMARC Record (Required for All Sending Domains)

1. Click **Create another record**
2. Select `TXT` as the **Type**
3. Enter `_dmarc.sender` for the **Name**
4. Enter `v=DMARC1;p=none;` for the **Value**
5. Save the settings

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `_dmarc.sender` |
| Value | Value | `v=DMARC1;p=none;` |

## Tracking Domain Setup

To set up a custom tracking domain, add two CNAME records:

### SSL Certificate CNAME

1. Click **Create another record**
2. Select `CNAME` as the **Type**
3. Copy the **Name** from your Lettr dashboard (e.g., `_acme-challenge.click.sender`)
4. Copy the **Value** from your Lettr dashboard
5. Save the settings

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `_acme-challenge.click.sender` |
| Value | Value | (provided in dashboard) |

### Traffic CNAME

1. Click **Create another record**
2. Select `CNAME` as the **Type**
3. Enter your tracking subdomain for the **Name** (e.g., `click.sender`)
4. Enter `proxy.lettr-tracking.com` for the **Value**
5. Save the settings

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `click.sender` |
| Value | Value | `proxy.lettr-tracking.com` |

<Tip>
  SSL certificates are automatically generated and renewed by Lettr. All tracking URLs use HTTPS.
</Tip>

## Inbound Domain Setup

To receive emails at your domain:

1. Create a new subdomain named `inbound` (or your chosen name)
2. Navigate to the subdomain's settings
3. Go to the **DNS** tab and click **manage** on MX records
4. Select **own mail server**

Add the following MX records:

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail server | Value | `rx1.sparkpostmail.com.` |
| Priority | Priority | `10` (Low) |

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail server | Value | `rx2.sparkpostmail.com.` |
| Priority | Priority | `10` (Low) |

| Strato | Lettr | Example Value |
|--------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail server | Value | `rx3.sparkpostmail.com.` |
| Priority | Priority | `10` (Low) |

<Info>
  Strato may require a trailing period on MX values. Add `.` at the end if verification fails.
</Info>

<Warning>
  When you configure MX records, all emails sent to that domain/subdomain will be routed to Lettr. We recommend using a subdomain like `inbound.example.com`.
</Warning>

## Complete Verification

Return to your Lettr dashboard and click **Verify**. DNS propagation typically takes a few hours.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Records not detected">
    Strato DNS may take several hours to propagate. Use [WhatsMyDNS](https://www.whatsmydns.net/) to check status.
  </Accordion>
  <Accordion title="Conflict with Strato mail server">
    For sending domains, create a subdomain to avoid conflicting with Strato's default mail configuration.
  </Accordion>
  <Accordion title="MX records require trailing period">
    Add a trailing period (`.`) to MX record values if verification fails.
  </Accordion>
</AccordionGroup>
