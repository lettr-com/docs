# Hetzner

This guide walks you through adding DNS records on Hetzner to verify your domain with Lettr.

<Tip>
  It is best practice to use a subdomain (e.g., `sender.example.com`) instead of the root domain. Using a subdomain allows for proper reputation segmentation and is especially important if receiving emails with Lettr.
</Tip>

## Add Domain to Lettr

First, log in to your [Lettr Dashboard](https://app.lettr.com) and go to **Domains â†’ Add Domain**. Enter your domain name and select the domain type.

## Log in to Hetzner

1. Log in to your [Hetzner DNS Console](https://dns.hetzner.com)
2. Select your domain from the **Your Zones** list
3. Navigate to the **Records** tab to manage DNS records

---

## Sending Domain Setup

Choose one of the following options based on whether you're using a subdomain or primary domain.

### Option 1: Subdomain (CNAME Verification)

If you're setting up a subdomain like `sender.example.com`, add the following records:

#### Bounce Domain CNAME

1. In the **Create Record** section, select `CNAME` as the **Type**
2. Enter `sender` for the **Name** (or your chosen subdomain)
3. Enter `sparkpostmail.com` for the **Value**
4. Set **TTL** to `1800`
5. Click **Add Record**

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `sender` |
| Value | Value | `sparkpostmail.com` |
| TTL | - | `1800` |

### Option 2: Primary Domain (TXT SPF Verification)

If you're using your root domain `example.com`, add an SPF record:

#### SPF Record

1. In the **Create Record** section, select `TXT` as the **Type**
2. Enter `@` for the **Name** (root domain)
3. Enter `v=spf1 include:_spf.sparkpostmail.com ~all` for the **Value**
4. Set **TTL** to `1800`
5. Click **Add Record**

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `@` |
| Value | Value | `v=spf1 include:_spf.sparkpostmail.com ~all` |
| TTL | - | `1800` |

<Info>
  If you already have an SPF record, merge Lettr's include with your existing record.
</Info>

### DKIM Record (Required for All Sending Domains)

1. Select `TXT` as the **Type**
2. Copy the **Name** from your Lettr dashboard (e.g., `scph0226._domainkey.sender`)
3. Copy the **Value** from your Lettr dashboard
4. Set **TTL** to `1800`
5. Click **Add Record**

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `scph0226._domainkey.sender` |
| Value | Value | `v=DKIM1;k=rsa;h=sha256;p=MIGfMA0...` |
| TTL | - | `1800` |

<Info>
  The DKIM hostname and value are unique to your domain. Copy them exactly from your Lettr dashboard.
</Info>

### DMARC Record (Required for All Sending Domains)

1. Select `TXT` as the **Type**
2. Enter `_dmarc.sender` for the **Name**
3. Enter `v=DMARC1;p=none;` for the **Value**
4. Set **TTL** to `1800`
5. Click **Add Record**

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `_dmarc.sender` |
| Value | Value | `v=DMARC1;p=none;` |
| TTL | - | `1800` |

## Tracking Domain Setup

To set up a custom tracking domain, add two CNAME records:

### SSL Certificate CNAME

1. Select `CNAME` as the **Type**
2. Copy the **Name** from your Lettr dashboard (e.g., `_acme-challenge.click.sender`)
3. Copy the **Value** from your Lettr dashboard
4. Click **Add Record**

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `_acme-challenge.click.sender` |
| Value | Value | (provided in dashboard) |

### Traffic CNAME

1. Select `CNAME` as the **Type**
2. Enter your tracking subdomain for the **Name** (e.g., `click.sender`)
3. Enter `proxy.lettr-tracking.com` for the **Value**
4. Click **Add Record**

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `click.sender` |
| Value | Value | `proxy.lettr-tracking.com` |

<Tip>
  SSL certificates are automatically generated and renewed by Lettr. All tracking URLs use HTTPS.
</Tip>

## Inbound Domain Setup

To receive emails at your domain, add MX records:

1. Select `MX` as the **Type**
2. Enter `inbound` for the **Name**
3. Enter `rx1.sparkpostmail.com` for the **Mail server**
4. Enter `10` for the **Priority**
5. Click **Add Record**

Repeat for the other two MX records:

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail server | Value | `rx1.sparkpostmail.com` |
| Priority | Priority | `10` |

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail server | Value | `rx2.sparkpostmail.com` |
| Priority | Priority | `10` |

| Hetzner | Lettr | Example Value |
|---------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail server | Value | `rx3.sparkpostmail.com` |
| Priority | Priority | `10` |

<Warning>
  When you configure MX records, all emails sent to that domain/subdomain will be routed to Lettr. We recommend using a subdomain like `inbound.example.com`.
</Warning>

## Complete Verification

Return to your Lettr dashboard and click **Verify**. DNS propagation typically takes a few minutes to a few hours.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Records not detected">
    Hetzner DNS may take up to a few hours to propagate. Use [WhatsMyDNS](https://www.whatsmydns.net/) to check status.
  </Accordion>
  <Accordion title="MX record requires trailing period">
    Hetzner may require a trailing period on MX values (e.g., `rx1.sparkpostmail.com.`). Try adding it if verification fails.
  </Accordion>
</AccordionGroup>
