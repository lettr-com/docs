# Hostinger

This guide walks you through adding DNS records on Hostinger to verify your domain with Lettr.

<Tip>
  It is best practice to use a subdomain (e.g., `sender.example.com`) instead of the root domain. Using a subdomain allows for proper reputation segmentation and is especially important if receiving emails with Lettr.
</Tip>

## Add Domain to Lettr

Log in to your [Lettr Dashboard](https://app.lettr.com), go to **Domains**, open the tab for your domain type (e.g. **Sending Domains**), and click **+ New Sending Domain** (or the equivalent for Tracking, Inbound, or Storage). Enter your domain name.

## Log in to Hostinger

1. Log in to your [Hostinger account](https://auth.hostinger.com/login)
2. Navigate to **Domains** in the top menu
3. Select your domain from the **Domain portfolio** list
4. Go to **DNS / Nameservers** to manage DNS records

## Sending Domain Setup

Choose one of the following options based on whether you're using a subdomain or primary domain.

### Option 1: Subdomain (CNAME Verification)

If you're setting up a subdomain like `sender.example.com`, add the following records:

#### Bounce Domain CNAME

1. Set the **Type** to `CNAME`
2. Enter `sender` for the **Name** (or your chosen subdomain)
3. Enter `sparkpostmail.com` for the **Target**
4. Set **TTL** to `3600`
5. Click **Add Record**

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `sender` |
| Target | Value | `sparkpostmail.com` |
| TTL | - | `3600` |

### Option 2: Primary Domain (TXT SPF Verification)

If you're using your root domain `example.com`, add an SPF record:

#### SPF Record

1. Set the **Type** to `TXT`
2. Enter `@` for the **Name** (root domain)
3. Enter `v=spf1 include:_spf.sparkpostmail.com ~all` for the **TXT value**
4. Set **TTL** to `3600`
5. Click **Add Record**

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `@` |
| TXT value | Value | `v=spf1 include:_spf.sparkpostmail.com ~all` |
| TTL | - | `3600` |

<Info>
  If you already have an SPF record, merge Lettr's include with your existing record.
</Info>

### DKIM Record (Required for All Sending Domains)

1. Set the **Type** to `TXT`
2. Copy the **Name** from your Lettr dashboard (e.g., `scph0226._domainkey.sender`)
3. Copy the **TXT value** from your Lettr dashboard
4. Set **TTL** to `3600`
5. Click **Add Record**

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `scph0226._domainkey.sender` |
| TXT value | Value | `v=DKIM1;k=rsa;h=sha256;p=MIGfMA0...` |
| TTL | - | `3600` |

<Info>
  The DKIM hostname and value are unique to your domain. Copy them exactly from your Lettr dashboard.
</Info>

### DMARC Record (Required for All Sending Domains)

1. Set the **Type** to `TXT`
2. Enter `_dmarc.sender` for the **Name**
3. Enter `v=DMARC1;p=none;` for the **TXT value**
4. Set **TTL** to `3600`
5. Click **Add Record**

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `TXT` |
| Name | Host | `_dmarc.sender` |
| TXT value | Value | `v=DMARC1;p=none;` |
| TTL | - | `3600` |

## Tracking Domain Setup

To set up a custom tracking domain, add two CNAME records:

### SSL Certificate CNAME

1. Set the **Type** to `CNAME`
2. Copy the **Name** from your Lettr dashboard (e.g., `_acme-challenge.click.sender`)
3. Copy the **Target** from your Lettr dashboard
4. Click **Add Record**

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `_acme-challenge.click.sender` |
| Target | Value | (provided in dashboard) |

### Traffic CNAME

1. Set the **Type** to `CNAME`
2. Enter your tracking subdomain for the **Name** (e.g., `click.sender`)
3. Enter `proxy.lettr-tracking.com` for the **Target**
4. Click **Add Record**

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `CNAME` |
| Name | Host | `click.sender` |
| Target | Value | `proxy.lettr-tracking.com` |

<Tip>
  SSL certificates are automatically generated and renewed by Lettr. All tracking URLs use HTTPS.
</Tip>

## Inbound Domain Setup

To receive emails at your domain, add MX records:

1. Set the **Type** to `MX`
2. Enter `inbound` for the **Name**
3. Enter `rx1.sparkpostmail.com` for the **Mail Server**
4. Enter `10` for the **Priority**
5. Click **Add Record**

Repeat for the other two MX records:

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail Server | Value | `rx1.sparkpostmail.com` |
| Priority | Priority | `10` |

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail Server | Value | `rx2.sparkpostmail.com` |
| Priority | Priority | `10` |

| Hostinger | Lettr | Example Value |
|-----------|-------|---------------|
| Type | Type | `MX` |
| Name | Host | `inbound` |
| Mail Server | Value | `rx3.sparkpostmail.com` |
| Priority | Priority | `10` |

<Warning>
  When you configure MX records, all emails sent to that domain/subdomain will be routed to Lettr. We recommend using a subdomain like `inbound.example.com`.
</Warning>

## Complete Verification

Return to your Lettr dashboard and click **Verify**. DNS propagation typically takes a few hours but is often faster.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Records not detected">
    Hostinger DNS may take up to 24 hours to propagate. Use [WhatsMyDNS](https://www.whatsmydns.net/) to check status.
  </Accordion>
  <Accordion title="CNAME conflict">
    Remove any existing A or CNAME records for the same hostname before adding new records.
  </Accordion>
</AccordionGroup>
